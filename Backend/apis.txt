================================================================================
          DANH SÁCH CHI TIẾT CÁC API HỆ THỐNG QUẢN LÝ TÀI LIỆU NỘI BỘ VIETTEL
                       Thư mục nguồn: /Backend/routers
================================================================================

Tài liệu này tổng hợp toàn bộ các API được định nghĩa trong các Router của Backend,
bao gồm thông tin chi tiết về Đường dẫn (Path), Phương thức (HTTP Method), 
Hàm xử lý (Function), Dữ liệu đầu vào (Input) và Dữ liệu đầu ra (Output).

--- CÁC LOẠI PHÂN QUYỀN TRUY CẬP (DEPENDENCIES):
- get_current_user: Yêu cầu JWT token hợp lệ từ Header (Authorization: Bearer <token>)
- get_current_active_admin: Yêu cầu JWT token hợp lệ của tài khoản quản trị viên (Admin)
- get_current_unit_manager: Yêu cầu JWT token hợp lệ của quản lý đơn vị phòng ban (Manager)

================================================================================
1. AUTH ROUTER (auth_router.py)
   Prefix: /api/auth | Tag: Xác thực
================================================================================

API 1.1: ĐĂNG NHẬP HỆ THỐNG
--------------------------------------------------------------------------------
* API Function: login
* HTTP Method:  POST
* Path:         /api/auth/login
* Summary:      Đăng nhập hệ thống và trả về JWT Access Token.
* Input:
  - Body (JSON): LoginPayload
    + username (str): Tên đăng nhập (Bắt buộc, Tối thiểu 1 ký tự)
    + password (str): Mật khẩu (Bắt buộc, Tối thiểu 1 ký tự)
* Output:
  - Response Model: TokenResponse
    + access_token (str): JWT Access Token dùng cho các API sau
    + token_type (str): Loại token (Mặc định: "bearer")
  - HTTP Status: 200 OK

API 1.2: ĐỔI MẬT KHẨU NGƯỜI DÙNG
--------------------------------------------------------------------------------
* API Function: change_password
* HTTP Method:  PUT
* Path:         /api/auth/change-password
* Summary:      Đổi mật khẩu người dùng (Dùng chung cho mọi vai trò).
* Input:
  - Header: Authorization: Bearer <token> (Yêu cầu đăng nhập - get_current_user)
  - Body (JSON): ChangePasswordPayload
    + old_password (str): Mật khẩu hiện tại (Bắt buộc, Tối thiểu 1 ký tự)
    + new_password (str): Mật khẩu mới (Bắt buộc, Từ 6 đến 100 ký tự)
* Output:
  - Response (JSON):
    + message (str): "Đổi mật khẩu thành công. Vui lòng sử dụng mật khẩu mới cho lần đăng nhập sau."
  - HTTP Status: 200 OK

API 1.3: ĐĂNG XUẤT HỆ THỐNG
--------------------------------------------------------------------------------
* API Function: logout
* HTTP Method:  POST
* Path:         /api/auth/logout
* Summary:      Đăng xuất tài khoản (Client xóa JWT Token).
* Input:        Không có đầu vào
* Output:
  - Response (JSON):
    + message (str): "Đăng xuất thành công"
  - HTTP Status: 200 OK

3. ADMIN ROUTER (admin_router.py)
   Prefix: /api/admin | Tag: Admin
   (Yêu cầu quyền Admin tối cao - get_current_active_admin cho toàn bộ Router)
================================================================================

API 3.1: TẠO ĐƠN VỊ MỚI
--------------------------------------------------------------------------------
* API Function: create_unit
* HTTP Method:  POST
* Path:         /api/admin/units
* Summary:      Admin tạo đơn vị mới trong cây tổ chức.
* Input:
  - Header: Authorization: Bearer <token> (Quyền Admin)
  - Body (JSON): UnitCreate
    + name (str): Tên đơn vị (Bắt buộc, 1 - 255 ký tự)
    + parent_id (int, optional): ID đơn vị cha (Mặc định null nếu là đơn vị gốc)
    + quota_bytes (int): Hạn mức dung lượng đơn vị (bytes, Mặc định: 0)
* Output:
  - Response Model: UnitResponse
    + id (UUID): Mã định danh UUID của đơn vị
    + name (str): Tên đơn vị
    + parent_id (UUID, optional): ID đơn vị cha
    + path (str): Đường dẫn nhánh tổ chức
    + level (int): Cấp độ phân nhánh
    + quota_bytes (int): Hạn mức dung lượng của đơn vị
    + used_bytes (int): Dung lượng đơn vị đã sử dụng
    + created_at (datetime): Thời gian tạo
    + updated_at (datetime): Thời gian cập nhật
  - HTTP Status: 201 Created

API 3.2: LẤY CÂY ĐƠN VỊ
--------------------------------------------------------------------------------
* API Function: get_unit_tree
* HTTP Method:  GET
* Path:         /api/admin/units/tree
* Summary:      Lấy toàn bộ cây đơn vị hoặc cây con từ root_id.
* Input:
  - Header: Authorization: Bearer <token> (Quyền Admin)
  - Query Parameter:
    + root_id (int, optional): ID đơn vị gốc để lấy nhánh con. Mặc định lấy cả cây.
* Output:
  - Response Model (JSON đệ quy): UnitTreeNode
    + id (int): ID số
    + name (str): Tên đơn vị
    + level (int): Cấp độ
    + quota_bytes (int): Hạn mức đơn vị (bytes)
    + used_bytes (int): Dung lượng đơn vị đã dùng (bytes)
    + children (List[UnitTreeNode]): Danh sách các đơn vị con kế thừa dưới dạng cây
  - HTTP Status: 200 OK

API 3.3: DI CHUYỂN ĐƠN VỊ

API 3.5: TẠO NGƯỜI DÙNG MỚI
--------------------------------------------------------------------------------
* API Function: create_user
* HTTP Method:  POST
* Path:         /api/admin/users
* Summary:      Admin tạo tài khoản người dùng mới.
* Input:
  - Header: Authorization: Bearer <token> (Quyền Admin)
  - Body (JSON): UserCreate
    + username (str): Tên đăng nhập (Bắt buộc, 3 - 100 ký tự)
    + email (EmailStr, optional): Địa chỉ email
    + password (str): Mật khẩu (Bắt buộc, Tối thiểu 6 ký tự)
    + full_name (str, optional): Họ và tên đầy đủ (Tối đa 255 ký tự)
    + role (str): Vai trò: "admin", "manager", "user" (Mặc định: "user")
    + unit_id (UUID): Mã UUID phòng ban trực thuộc
    + quota_bytes (int): Hạn mức dung lượng cá nhân (bytes, Mặc định: 0)
* Output:
  - Response Model: UserResponse
    + id (UUID): ID người dùng (user_id)
    + username (str): Tên đăng nhập
    + full_name (str): Họ và tên đầy đủ
    + role (str): Vai trò tài khoản
    + unit_id (UUID, optional): ID đơn vị trực thuộc
    + quota_bytes (int): Hạn mức dung lượng cá nhân
    + used_bytes (int): Dung lượng cá nhân đã dùng
    + is_active (bool): Trạng thái tài khoản
    + created_at (datetime): Thời gian tạo
    + updated_at (datetime): Thời gian cập nhật
  - HTTP Status: 201 Created

API 4.2: UPLOAD TÀI LIỆU VÙNG CHUNG ĐƠN VỊ
--------------------------------------------------------------------------------
* API Function: upload_unit_document
* HTTP Method:  POST
* Path:         /api/manager/documents/upload
* Summary:      Upload tài liệu lên vùng lưu trữ dùng chung của phòng ban/đơn vị.
* Input:

    + owner_id (UUID): ID người upload (Manager)
    + unit_id (UUID): ID đơn vị sở hữu tài liệu
    + is_public (bool): Trạng thái công khai của tài liệu
    + created_at (datetime): Ngày giờ upload
    + folder_id (UUID, optional): ID thư mục cha nếu có

API 5.1: TRANG CHÍNH - DANH SÁCH TÀI LIỆU PHÂN QUYỀN
--------------------------------------------------------------------------------
* API Function: get_documents
* HTTP Method:  GET
* Path:         /api/documents/
* Summary:      Lấy toàn bộ danh sách tài liệu mà user có quyền truy cập.
                (Gộp từ 3 nguồn: cá nhân sở hữu, được chia sẻ, kế thừa từ đơn vị).
* Input:
  - Header: Authorization: Bearer <token> (Yêu cầu đăng nhập - get_current_user)
  - Query Parameters:
    + page (int): Số trang hiện tại (Mặc định: 1, tối thiểu 1)
    + page_size (int): Số bản ghi mỗi trang (Mặc định: 20, tối thiểu 1, tối đa 100)
* Output:
  - Response Model: DocumentListResponse
    + items (CategorizedDocumentItems):
      * personal (List[DocumentResponse]): Tài liệu do user tự sở hữu
      * shared (List[DocumentResponse]): Tài liệu được đồng nghiệp chia sẻ trực tiếp
      * unit_inherited (List[DocumentResponse]): Tài liệu kế thừa công khai từ phòng ban
    + totals (CategorizedDocumentCount):
      * personal (int): Tổng số lượng tài liệu cá nhân
      * shared (int): Tổng số lượng tài liệu được chia sẻ
      * unit_inherited (int): Tổng số lượng tài liệu kế thừa phòng ban
    + page (int): Số trang hiện tại
    + page_size (int): Kích thước trang
    + total_pages (int): Tổng số trang tương ứng
  - HTTP Status: 200 OK

API 5.3: CHIA SẺ TÀI LIỆU NGANG HÀNG
--------------------------------------------------------------------------------
* API Function: share_document
* HTTP Method:  POST
* Path:         /api/documents/{document_id}/share
* Summary:      Chia sẻ quyền đọc tài liệu cho một nhân sự bất kỳ trong cơ quan.
* Input:
  - Header: Authorization: Bearer <token> (Yêu cầu đăng nhập - get_current_user)
  - Path Parameter:
    + document_id (UUID): ID tài liệu cần chia sẻ
  - Body (JSON): DocumentSharePayload
    + username (str): Tên đăng nhập của người nhận chia sẻ (Bắt buộc)
    + unit_id (UUID): Mã UUID phòng ban của người nhận chia sẻ (Bắt buộc)
* Output:
  - Response Model: DocumentShareResponse
    + message (str): "Chia sẻ tài liệu thành công."
    + share_id (UUID): ID của lượt chia sẻ vừa tạo
    + document_id (UUID): ID tài liệu được chia sẻ
    + shared_by (UUID): ID người chia sẻ
    + shared_with_user_id (UUID): ID của người nhận chia sẻ
    + shared_at (datetime): Thời gian thực hiện chia sẻ
  - HTTP Status: 201 Created

API 5.4: USER UPLOAD TÀI LIỆU
--------------------------------------------------------------------------------
* API Function: user_upload_document
* HTTP Method:  POST
* Path:         /api/documents/upload
* Summary:      Người dùng upload tài liệu cá nhân, mặc định công khai nội bộ phòng ban.
* Input:
  - Header: Authorization: Bearer <token> (Yêu cầu đăng nhập - get_current_user)
  - Content-Type: multipart/form-data
  - Form Data:
    + file (UploadFile): Tệp tin cần tải lên (Bắt buộc)
    + title (str, optional): Tiêu đề hiển thị cho tài liệu (Nếu trống, lấy tên file)
    + folder_id (UUID, optional): ID thư mục cha nếu tệp thuộc một folder
* Output:
  - Response Model: DocumentResponse (Xem chi tiết trường tại API 4.2)
  - HTTP Status: 200 OK
